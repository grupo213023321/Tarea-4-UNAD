# Sistema de Gestion integral - Software FJ
## Proyecto colaborativo Fase 4 Programación UNAD


Este sistema gestiona clientes, servicios y reservas sin el uso de bases de datos externas.
El sistema está diseñado bajo el paradigma de Programación Orientada a Objetos (POO), implementando una arquitectura sin bases de datos que prioriza la estabilidad y el manejo avanzado de errores.

## CARACTERÍSTICAS:

- Uso de clases
- Abstración Herencia y Polimorfismo
- Manejo de Excepciones
- Uso de Logs

## ARQUITECTURA:

El diseño de este software se basa en la modularidad, que permite separar partes del programa para depurar y tener facilidad de mantenmiento
el archivo classes.py contiene todas las clases que se usan en el programa, asi como interfaz.py aloja unicamente las funciones relacionadas con la interfaz.
exepciones.py, servicios.py, log.py, validadores.py lo en conjunto se encargan de gestionar las diferentes funciones del programa por modulos como cada uno de sus nombres indica.

## USO:
### ¡Para iniciar el programa se usa el archivo main.py!
